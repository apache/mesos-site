27 // The dispatch mechanism enables you to "schedule" a method to get
28 // invoked on a process. The result of that method invocation is
29 // accessible via the future that is returned by the dispatch method
30 // (note, however, that it might not be the _same_ future as the one
31 // returned from the method, if the method even returns a future, see
32 // below). Assuming some class 'Fibonacci' has a (visible) method
33 // named 'compute' that takes an integer, N (and returns the Nth
34 // fibonacci number) you might use dispatch like so:
35 //
36 // PID<Fibonacci> pid = spawn(new Fibonacci(), true); // Use the GC.
37 // Future<int> f = dispatch(pid, &Fibonacci::compute, 10);
38 //
39 // Because the pid argument is "typed" we can ensure that methods are
40 // only invoked on processes that are actually of that type. Providing
41 // this mechanism for varying numbers of function types and arguments
42 // requires support for variadic templates, slated to be released in
43 // C++11. Until then, we use the Boost preprocessor macros to
44 // accomplish the same thing (albeit less cleanly). See below for
45 // those definitions.
46 //
47 // Dispatching is done via a level of indirection. The dispatch
48 // routine itself creates a promise that is passed as an argument to a
49 // partially applied 'dispatcher' function (defined below). The
50 // dispatcher routines get passed to the actual process via an
51 // internal routine called, not suprisingly, 'dispatch', defined
52 // below:
53

56 // The internal dispatch routine schedules a function to get invoked
57 // within the context of the process associated with the specified pid
58 // (first argument), unless that process is no longer valid. Note that
59 // this routine does not expect anything in particular about the
60 // specified function (second argument). The semantics are simple: the
61 // function gets applied/invoked with the process as its first
62 // argument. Currently we wrap the function in a shared_ptr but this
63 // will probably change in the future to unique_ptr (or a variant).
64 void dispatch(
65  const UPID& pid,
66  const std::shared_ptr<std::function<void(ProcessBase*)>>& f,
67  const Option<const std::type_info*>& functionType = None());
