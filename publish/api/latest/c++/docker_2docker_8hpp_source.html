134  // Performs 'docker run IMAGE'. Returns the exit status of the
135  // container. Note that currently the exit status may correspond
136  // to the exit code from a failure of the docker client or daemon
137  // rather than the container. Docker >= 1.10 [1] uses the following
138  // exit statuses inherited from 'chroot':
139  // 125 if the error is with Docker daemon itself.
140  // 126 if the contained command cannot be invoked.
