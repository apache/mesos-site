27 // The `nvml` namespace acts as a higher-level wrapper on top of the
28 // NVIDIA Management Library (NVML). By providing this higher-level
29 // API, we eliminate the build-time dependency on `libnvidia-ml` in
30 // favor of a runtime dependency. The caller may probe to see if the
31 // library is available, and may take different code paths accordingly.
32 // This is advantageous for deploying a common `libmesos` on machines
33 // both with and without GPUs installed. Only those machines that have
34 // GPUs need to have the `libnvidia-ml` library installed on their
35 // system.
36 //
37 // TODO(klueska): Add unit tests and provide finalization support.
