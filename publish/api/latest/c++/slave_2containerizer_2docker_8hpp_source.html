427  // The DockerContainerizer needs to be able to properly clean up
428  // Docker containers, regardless of when they are destroyed. For
429  // example, if a container gets destroyed while we are fetching,
430  // we need to not keep running the fetch, nor should we try and
431  // start the Docker container. For this reason, we've split out
432  // the states into:
433  //
434  // FETCHING
435  // PULLING
436  // MOUNTING
437  // RUNNING
438  // DESTROYING
439  //
440  // In particular, we made 'PULLING' be it's own state so that we
441  // can easily destroy and cleanup when a user initiated pulling
442  // a really big image but we timeout due to the executor
443  // registration timeout. Since we currently have no way to discard
444  // a Docker::run, we needed to explicitly do the pull (which is
445  // the part that takes the longest) so that we can also explicitly
446  // kill it when asked. Once the functions at Docker::* get support
447  // for discarding, then we won't need to make pull be it's own
448  // state anymore, although it doesn't hurt since it gives us
449  // better error messages.
450  enum State
451  {
452  FETCHING = 1,
453  PULLING = 2,
454  MOUNTING = 3,
455  RUNNING = 4,
456  DESTROYING = 5
457  } state;
