1011  private:
1012  // Heuristically tries to determine whether a quota request could
1013  // reasonably be satisfied given the current cluster capacity. The
1014  // goal is to determine whether a user may accidentally request an
1015  // amount of resources that would prevent frameworks without quota
1016  // from getting any offers. A force flag will allow users to bypass
1017  // this check.
1018  //
1019  // The heuristic tests whether the total quota, including the new
1020  // request, does not exceed the sum of non-static cluster resources,
1021  // i.e. the following inequality holds:
1022  // total - statically reserved >= total quota + quota request
1023  //
1024  // Please be advised that:
1025  // * It is up to an allocator how to satisfy quota (for example,
1026  // what resources to account towards quota, as well as which
1027  // resources to consider allocatable for quota).
1028  // * Even if there are enough resources at the moment of this check,
1029  // agents may terminate at any time, rendering the cluster under
1030  // quota.
1031  Option<Error> capacityHeuristic(
1032  const mesos::quota::QuotaInfo& request) const;
1033 
1034  // We always want to rescind offers after the capacity heuristic. The
1035  // reason for this is the race between the allocator and the master:
1036  // it can happen that there are not enough free resources at the
1037  // allocator's disposal when it is notified about the quota request,
1038  // but at this point it's too late to rescind.
1039  //
1040  // While rescinding, we adhere to the following rules:
1041  // * Rescind at least as many resources as there are in the quota request.
1042  // * Rescind all offers from an agent in order to make the potential
1043  // offer bigger, which increases the chances that a quota'ed framework
1044  // will be able to use the offer.
1045  // * Rescind offers from at least `numF` agents to make it possible
1046  // (but not guaranteed, due to fair sharing) that each framework in
1047  // the role for which quota is set gets an offer (`numF` is the
1048  // number of frameworks in the quota'ed role). Though this is not
1049  // strictly necessary, we think this will increase the debugability
1050  // and will improve user experience.
1051  //
1052  // TODO(alexr): Consider removing this function once offer management
1053  // (including rescinding) is moved to allocator.
1054  void rescindOffers(const mesos::quota::QuotaInfo& request) const;
