31 // NOTE: We delete these functions because they are not meaningful on Windows.
32 // `su` and `user` are the most important of these functions. The POSIX code
33 // uses them prodigiously, but in Windows we have been able to divest ourselves
34 // of all uses.
35 //
36 // `su` is important to the launcher API; if the `user` flag (not to be
37 // confused with the `user` function, which we delete below) is present, we
38 // will `su` to that user before launching the command. On Windows we avoid
39 // this problem by simply conditionally compiling out the `user` flag
40 // altogether, which means that we never have to call `su`.
41 //
42 // The `user` function itself is already mostly conditionally compiled out of
43 // every platform except linux. So in this case it is simply safe to return an
44 // error on Windows.
